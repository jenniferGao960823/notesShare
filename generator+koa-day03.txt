nodemon  热重载-》  node中的启动工具、用来启动脚手架
下载npm install nodemon --save
在package.json文件中的scripts下任意命名一个属性，属性值为nodemon 路径
启动方式：在cmd中输入=》npm run 属性名   (start属性不需要用run启动，因为start本身带有启动的意思)
热重载的作用：不用每次都启动服务器，会自动监视状态

supervisor  和nodemon一样

脚手架：(生成器)  可以理解为一个毛坯房
1、npm install express-generator -g 安装express脚手架
express --version 查看版本
2、express --view=ejs app 创建一个app文件夹  文件名可以更改
3、cd app 进入app目录
4、npm i  安装依赖          生成node-modules文件夹
5、SET DEBUG=app:* & npm start  运行app
6、npm run dev 启动脚手架，需要在package.json中的scripts里写入dev的路径

!!!给别人发项目不要把node-modules放进去，不但使项目很大，执行时还会报错

koa： express的原班人马开发
定义：基于nodejs的下一代web开发框架
koa置入了es6语法   async/await  异步加载  返回的是promise对象（resolve，reject）
下载：npm install koa --save
koa引入后需要通过new实例化

koa-views  加载视图（引擎）
var views=require('koa-views')

配置ejs引擎
方法一：使用选中视图（ejs）  文件的后缀名为html
 app.use(views(__dirname+'/views',{
     map:{html:'ejs'}                //map映射   把html文件映射为ejs文件（ejs当html去解析）
 }))
方法二：文件的后缀名为ejs
app.use(views('views',{
    extension:'ejs'
}))

art-template模板引擎
var art=require('koa-art-template');
配置koa-art-template模板引擎
art(app,{
    root:path.join(__dirname,'views'),    //视图位置
    extname:'.html',      //后缀名 （.ejs）
    debug:process.env.NODE_ENV!=='production'     //是否开启调试模式
})

koa中的ctx==》context上下文  内部封装了request和response，ctx.body等价于res.send
ctx.request.query获取传输字段
